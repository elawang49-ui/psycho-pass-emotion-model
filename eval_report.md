## 主要迭代过程

1. 中文短句误判修复
   - 早期模型对“我喜欢你”“我真的很开心”等短文本存在明显误判。
   - 通过补充短句样本、改进向量化方式和增加兜底规则改善。

2. 字符级 ngram
   - 将默认 TF-IDF 改为字符级 ngram。
   - 适配中文无天然空格的特点，提高短文本泛化能力。

3. 极短词兜底
   - 为“开心”“喜欢”“安心”“烦”“好累”等极短词加入规则分数。
   - 避免传统模型在短输入下出现空特征或弱特征误判。

4. 反讽兜底
   - 为“挺好的，下次别做了”“我真的谢谢你啊”等反讽句加入规则兜底。
   - 显示为“阴阳怪气 / 轻微负向”，减少被误判为普通愤怒或正向。

5. 低唤醒负面提升
   - 补充疲惫、无力、失望、麻木等低唤醒负面样本。
   - 低唤醒负面通过率得到阶段性提升。

6. 网感语料加入
   - 加入更贴近中文网络表达的短句和口语化样本。
   - 提升部分高唤醒正面与反讽表现，但也带来分布牵引，需要继续平衡。

7. 中性数据修复
   - 补充纯中性 / 低波动日常陈述后，中性日常陈述评估恢复到较高水平。

8. 数据清洗与无效标签隔离
   - 新增 `clean_dataset.py`。
   - 将非数值 `valence/arousal` 样本保存到 `data_invalid_rows.csv`，训练只使用 `data_clean.csv`。

9. badcase 分析与定向补数据
   - 新增 `analyze_test_failures.py`。
   - 按 case_type 分析 valence / arousal 偏高或偏低，为后续补数据提供方向。

## 当前局限

- 明确正向短句、高唤醒正面、混合复杂情绪仍有较大提升空间。
- 模型对个别高唤醒正向句仍可能低估 valence 或 arousal。
- 明确负向短句中，部分高唤醒负面与低唤醒负面仍会互相牵引。
- 当前评估集规模有限，结果仅作为阶段性参考。

## 在线大模型使用说明

部分训练语料由在线大模型辅助生成，但最终预测模型为本地轻量机器学习模型。

最终训练和推理流程使用 `TF-IDF 字符级 ngram + MultiOutputRegressor + RandomForestRegressor`，运行时不调用在线大模型 API，不依赖 transformers 或 HuggingFace。
